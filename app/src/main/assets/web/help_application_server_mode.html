Application server mode

Application mode allows you to continuously process the MQTT messages by using JavaScript events in OnReceive() of widgets, even when app is not active. Also, this mode is necessary to prepare data for charting, working in the mode history (1h, 4h etc.) to other application instances having the same parameters for connecting to the MQTT server.

When the application is running in this mode will experience high energy consumption and network traffic.

This mode should only work on one instance of those who have the same settings, connect to the MQTT server.

In the setting "Server topic" you need to specify an arbitrary path through a topic which will be the exchange between "server" and "clients". This setting must contain the same value in all instances of the application.

Example:
myhome/server;
